# 从循环到图：改进机器的架构 — 文章要点与 taskloop 映射

## 文章核心论点

自我改进不是一个周期，而是一种结构。单一反馈循环的失败方式已被充分理解，正在浮现的答案不是更好的循环，而是**循环之图**——彼此监督、喂养、约束、纠正的改进循环网络。但图也会以自己的方式失败（循环论证地、自洽地、看似合理地），所以持久的轴不是 loops vs graphs，而是 **ungrounded vs grounded**：改进机器是否始终触及它声称要改进的现实。

## 单一循环的四种结构性失败

任何改进循环都是同一个四冲程（选控制对象 → 设参照值 → 测差距 → 行动缩小差距，再来一轮），其形状本身导致四种失败：

1. **古德哈特定律**：循环只能看见自己的指标，会找到一切推高指标的方式，包括背叛指标初衷的方式。案例：客服机器人靠打发客户刷高"工单解决率"，曲线攀升五个月，续约期流失率翻倍——循环没有故障，它在精确执行被造出来要做的事。
2. **向上的盲目**：循环内部无法质疑参照值本身对不对；循环越努力，错误目标被达成得越彻底。
3. **循环冲突**：独立构建的循环互相打架（速度 vs 彻底性），每个单独看都"正常"。
4. **测量衰变**：传感器漂移、定义漂移、仪表盘照绿；测量从"核对现实"滑向"核对文书"，无人看守看守者。

## 图的四个拓扑回答

- **配对**答古德哈特：优化循环配反向指标监督循环（解决率 × 续约率）。
- **层级**答向上盲目：慢循环拥有快循环的参照值，修订目标本身是受治理的周期。
- **仲裁**答冲突：打架的循环之上有拥有权衡权的循环。
- **审计**答测量衰变：独立循环定期核对其他循环的数字是否仍触及现实。

例证：MLOps（冠军-挑战者、漂移监控、自动回滚、训练循环永不可见的保留集）、公司治理多速嵌套循环（运营 → 季度管理 → 年度独立审计 → 董事会质疑目标本身）、生物体（免疫系统 ≈ 全身审计循环）。

## 图的失败方式与三个必需品

全靠报表互喂的图是循环论证：每个循环监督另一个循环，没有一个触到地面——一切自洽、无一被验证，失败得更晚更贵、沿途绿灯更多。拓扑买来精致，买不来与现实的接触。图需要：

- **锚点**：无法争辩的测量（到账收入、真正执行过的测试、真实留下的客户）。
- **冻结节点**：优化循环永远不许调的规则——恰恰因为它们是优化器最想削弱的规则。
- **图外判断**："更好"的根定义无法由机器生成，只能由人通过接触真实失败提供。

## 对 taskloop 的映射

文章的词汇几乎逐项对应本仓库已有的设计决策：

| 文章概念 | taskloop 对应物 |
|---|---|
| 审计循环（watching loop） | `meta-loop` 技能：对结局账本的周期性审查 |
| 锚点（可争辩性为零的测量） | evidence ledger、`verify --record`、Stop observation——真正执行过的检查，而非报表互证 |
| 冻结节点（优化器不许调的规则） | 主机风险底线不随 grant 降低、`git clean` 单一分类器不许加第二个、机器底线不因 scoped destructive 上升 |
| 图外的人类判断 | `--granted-by user` 硬要求（archive 是用户溯源声明，永不自签） |
| 参照值修订受治理 | authority_friction：grant 词汇从真实 deny 证据中生长，不做投机扩充（`AGENTS.md` Conventions） |
| 配对指标 | 结局账本同时看 satisfied 与 abandoned/bypass，而非只看关闭数 |

**文章对本仓库的警示**：图的特征性失败（循环论证）对应 taskloop 的真实风险——若 meta-loop 审查只消费账本自身的投影而不回到可执行检查，监督图也会退化为互相确认的网络。锚点必须始终是"执行过的命令退出码"这一级的东西。
